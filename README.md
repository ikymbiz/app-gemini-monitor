# 📊 Gemini API Monitor (v4.0-Budget)

Gemini API（Google Cloud）の過去24時間の利用状況をモニタリングし、推定コストと予算の進捗をリアルタイムに視覚化するシングルページ・ダッシュボードです。

サーバーサイドの実装は不要で、ブラウザ（クライアントサイド）のみで完結して動作します。設定内容はブラウザのIndexedDBに自動保存されるため、毎回の入力の手間を省けます。

## ✨ 主な機能 (Features)

- 📈 **リアルタイム・トラフィック監視**
  - Google Cloud Monitoring APIと連携し、過去24時間のAPIリクエスト数と推移グラフ（Chart.js）を表示します。
- 💰 **高度な予算・コスト管理**
  - API単価（100万トークンあたりのUSD）と月間予算を設定し、推定利用コストを自動計算します。
  - 予算に対する進捗状況をプログレスバーで視覚化します。
- ⚠️ **アラート機能**
  - 使用量が設定した「警告閾値（%）」を超えると、UIが赤色に変化し視覚的に警告します。
- 💾 **設定の自動保存 (Auto-save)**
  - 入力したプロジェクト情報や予算設定は即座にブラウザの `IndexedDB` に保存され、次回アクセス時に自動で復元されます。
- 🔒 **セキュアな認証**
  - Google Identity Servicesを活用し、ブラウザから直接安全にOAuth認証を行います。

## 🛠 技術スタック (Tech Stack)

- **Frontend**: HTML5, Vanilla JavaScript
- **Styling**: Tailwind CSS (CDN経由)
- **Charts**: Chart.js (CDN経由)
- **Storage**: IndexedDB (設定の永続化)
- **API**: Google Cloud Monitoring API (`monitoring.read`), Google Identity Services

## 🚀 使い方 (How to Use)

本ツールはビルド不要です。以下の手順ですぐに利用を開始できます。

### 1. Google Cloud 側の事前準備

本ツールを使用するには、Google Cloud Project側で以下の設定が必要です。

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス。
2. 対象のプロジェクトで **Cloud Monitoring API** を有効化。
3. **「APIとサービス」>「認証情報」** へ移動し、「OAuth クライアント ID（ウェブ アプリケーション）」を作成。
4. 「承認済みの JavaScript オリジン」に、本ツールをホスティングするURL（ローカルの場合は `http://localhost` など）を追加。

### 2. ツールの起動と設定

1. リポジトリをクローンするか、`index.html` ファイルをダウンロードします。
2. ローカルサーバー（VSCodeのLive Server等）または任意のホスティング環境で `index.html` を開きます。
3. 画面の指示に従い、以下の設定を入力します。
   - **プロジェクト設定**: OAuth Client ID, GCP Project ID
   - **予算・単価設定**: 月間予算(USD), 警告閾値(%), 単価(USD/1M Token)
4. 入力は即座に自動保存されます。
5. 「**Googleでログイン**」ボタンをクリックし、モニタリング権限を許可するとダッシュボードが表示されます。

## 🧮 計算仕様について (Calculation Logic)

本ダッシュボードで表示される数値は、以下のロジックに基づいた**推定値**です。

- **推定トークン数**: `総リクエスト数 × 1,500 トークン`（※固定値による推計）
- **推定コスト**: `(推定トークン数 ÷ 1,000,000) × 設定した単価`

> [!WARNING]
> 本ツールで表示されるトークン数およびコストは推定値であり、Google Cloudの実際の請求ダッシュボードの数値と完全に一致するものではありません。正式な請求額は必ずGoogle Cloud Consoleにてご確認ください。
